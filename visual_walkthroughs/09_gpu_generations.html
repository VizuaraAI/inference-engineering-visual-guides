IMPACT

What This Means for Inference

Each GPU generation unlocks new capabilities for serving larger models faster and more efficiently.

Hopper (H100)

Enabled FP8 inference and the Transformer Engine. Made serving 70B parameter models on a single node practical. Established the standard for modern LLM inference.

Llama-2 70B: ~2 GPUs needed
Inference: ~100 tok/s (batch=1)

Blackwell (B200/GB200)

Introduces FP4 precision (2x throughput vs FP8). Dual-die design with 192 GB HBM3e. GB200 NVL72 creates a 13.8 TB memory domain for trillion-parameter models.

Llama-3 405B: single NVL72 rack
2.5x BW, 2.4x HBM vs H100

Rubin (2026)

HBM4 memory with unprecedented bandwidth. Vera ARM CPU replaces Grace. Designed for 10T+ parameter MoE models and real-time multimodal reasoning at scale.

Target: 10T+ parameter models
HBM4: >12 TB/s bandwidth

Key Trend: Memory Bandwidth Grows Faster Than Compute

Each generation roughly doubles compute TFLOPS, but memory bandwidth grows at an even faster rate (2-2.5x). This is intentional -- NVIDIA recognizes that LLM inference is memory-bandwidth-bound and is prioritizing bandwidth scaling to close the gap.
